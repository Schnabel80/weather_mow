# WeatherMow v0.4 — Unified Physical Wetness Model

## Kernidee

Alle Nässe-Quellen in einer physikalischen Einheit: **`wetness_mm`** (0–20 mm).
Trocknung über vereinfachtes Penman-Monteith: Solar + Temperatur (VPD) + Wind.

---

## Modell

### Zustandsvariable
```
wetness_mm  ∈ [0.0, WETNESS_MAX=20.0]   # persistiert via HA Storage + Recorder-Restore
```

### Pro 5-Min-Update
```
wetness_mm += rain_mm          # Regen-Delta aus Puffer
wetness_mm += condensation_mm  # Tau-Bildung bei VPD < DEW_OFFSET
wetness_mm -= drying_mm        # Penman-Trocknung
wetness_mm  = clamp(0, 20)
```

### Trocknungsformel
```
eff_solar       = solar_factor × lawn_sun_efficiency   (gated by lawn_sun_from)
drying_mm       = K_SOLAR × eff_solar
                + K_TEMP  × max(0, VPD)
                + K_WIND  × wind_kmh

condensation_mm = K_COND × max(0, DEW_OFFSET − VPD)   # Gras kühler als Luft
```

### Kalibrierungs-Konstanten (Provisorium, Beta-Validierung nötig)
| Konstante | Wert | Bedeutung |
|-----------|------|-----------|
| K_SOLAR | 0.030 mm/Update | Peak-Sonne → ~0.36 mm/h |
| K_TEMP | 0.001 mm/Update/°C | VPD=10°C → ~0.12 mm/h |
| K_WIND | 0.0005 mm/Update/(km/h) | 20 km/h → ~0.06 mm/h |
| K_COND | 0.003 mm/Update/°C | 3°C unter Taupunkt → ~0.22 mm/h |
| DEW_OFFSET | 3.0 °C | Grasoberfläche kühlt ~3°C unter Lufttemperatur |
| WETNESS_MAX | 20.0 mm | Physikalischer Deckel (Ablauf) |

### Mäh-Entscheidung
```
mow_allowed  wenn  wetness_mm < mow_threshold_mm
```

## Nässe-Quellen

| Quelle | Mechanismus | v0.3-Äquivalent |
|--------|-------------|-----------------|
| **Regen** | Delta rain_today_mm pro Update → direkt auf wetness_mm | rain_buffer × weights × 20 |
| **Tau** | K_COND wenn VPD < DEW_OFFSET → kontinuierlich aufbauen | Binärer 35-Punkte-Score |
| **Bewässerung** | Button bucht `irrigation_amount_mm` einmalig | irrigation_wetness_boost (70 Punkte) |

### Wegfallende Komponenten
- `rain_weighted_12h` → ersetzt durch direktes mm-Akkumulieren
- `morning_penalty` → fällt weg (organisch im Modell abgebildet)
- `dew_score` (binär 35 Punkte) → ersetzt durch kontinuierliche Kondensation
- `irrigation_wetness_boost` (Score-basiert) → ersetzt durch `irrigation_amount_mm`
- 0–100 Score → komplett ersetzt durch mm

## Migration v0.3 → v0.4 (Option C: Recorder-Rekonstruktion)

Beim ersten Start:
1. Rain-Buffer aus Recorder lesen → rain_weighted_12h berechnen → in mm umrechnen
2. Letzten `irrigation_boost`-Zustand aus Recorder lesen → ÷ 20 in mm
3. Tau-Status aus Recorder (dew_present) → Kondensations-mm schätzen
4. `wetness_mm = rain_mm + irrigation_mm + dew_mm`

Fallback wenn Recorder leer: `wetness_mm = 0` (sauberer Neustart).

## Mäh-Entscheidung & Forecast

### Adaptiver Startschwellwert

```python
good_forecast = rain_fc_3h == 0          # kein Regen in 3h (OWM, alle 5 Min aktuell)

if urgency_high or not good_forecast:
    effective_threshold = mow_threshold   # normal, kein Grace
    grace_period_active = False
else:
    effective_threshold = max(0.0, mow_threshold - 0.30)  # Discount 0.3mm (fix)
    grace_period_active = True            # 30min Grace (fix)
```

**Rationale rain_fc_3h:** Akku ~1h → 3h = ~2 Mähzyklen möglich. Update jede 5 Min
→ Mäher startet beim nächsten Update sobald Regen aus dem 3h-Fenster fällt.

**Jeder Tag ist unabhängig.** Kein Cross-Day-Forecast, kein Carryover der Entscheidung.

### Grace Period (30 Minuten)

```python
# In _compute_decision():
if wetness_mm < effective_threshold:
    if _below_threshold_since is None:
        _below_threshold_since = now
    if (now - _below_threshold_since) >= timedelta(minutes=30) or not grace_period_active:
        mow_allowed = True
        # status: mowing_allowed
    else:
        # status: waiting_for_favorable
else:
    _below_threshold_since = None
    # status: too_wet
```

### Status-Matrix

| good_forecast | wetness vs threshold | Grace | Status |
|---------------|---------------------|-------|--------|
| nein | > threshold | — | `too_wet` |
| nein | ≤ threshold | — | `mowing_allowed` |
| ja | > mow_threshold | — | `too_wet` |
| ja | eff_thr < wet ≤ mow_thr | — | `waiting_for_favorable` |
| ja | ≤ eff_threshold | läuft | `waiting_for_favorable` |
| ja | ≤ eff_threshold | abgelaufen | `mowing_allowed` |
| urgency_high | ≤ threshold | — | `mowing_allowed` (Override) |

### next_mow_expected — Penman-Projektion

OWM-Forecast liefert stündlich Temperatur + Strahlung + Regen:

```python
for h in forecast_hours_today:
    eff_solar_h = solar_h * efficiency
    vpd_h       = temp_h - dew_point_now   # DP näherungsweise konstant
    drying_h    = K_SOLAR * eff_solar_h + K_TEMP * max(0, vpd_h)
    rain_h      = precip_h
    wetness_h   = max(0, wetness_{h-1} + rain_h - drying_h)
    if wetness_h < effective_threshold:
        next_mow = h + (30min if grace_period_active else 0)
        break
```
